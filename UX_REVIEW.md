# UX Review — First-Time, Unguided User

**Package:** FEMA Program ID & PRA Automation (concept demo)
**Review date:** 2026-07-10
**Reviewer lens:** heuristic evaluation + first-time-user walkthrough. Two personas: (P1) a FEMA/consulting risk or financial analyst opening the file cold, with no training, who must leave with a defensible preliminary risk assessment; (P2) an executive/SME following the 10-screen story without narration.

## What was reviewed (Phase 0 — repo state)

- **The Streamlit UI (Wave 6 Option A) is NOT built.** No Streamlit/app code exists in the repo; the only `.py` files are the data generator (`data/generator/generate_synthetic.py`) and the leave-behind builder (`leavebehind/build_demo_html.py`). `12-implementation-roadmap.md:34` confirms Wave 6 Streamlit is "Planned."
- **A full 10-screen UI IS built** as the single-file HTML leave-behind: `leavebehind/fema-demo.html`, generated from `leavebehind/template.html` (the UI source of truth, per `leavebehind/README.md:80`). This review treats **the leave-behind as the primary built artifact** and the storyboard (`11-demo-storyboard.md`) + solution design as intent.
- **Method:** full read of `template.html` (all 2,681 lines — markup, microcopy, and JS renderers), the design docs (files 03, 05, 06, 10, 11, 12, 14), the demo script and README, and git history; plus the file was **opened and driven in a real browser** (screens 1, 3, 7, 8 observed at 1262×568). Findings below marked **[Observed]** were seen live or read directly in source; **[Assumption]** marks inferences about what a user would feel or do.
- Numeric correctness is **not** re-litigated here — `review/DEMO_GAP_ANALYSIS.md` §4 already verified the math independently (zero mismatches) and logged functional gaps GAP-01…14. This review is complementary (first-time-user UX) and cross-references GAP-IDs where they overlap.

## Executive summary

The demo is genuinely strong on the hardest trust problems — every number carries evidence, deterministic and AI outputs wear different badges, the follow-the-dollar trace and parity check make auditability a visible property, and the review/override/finalize flow enforces human-in-the-loop mechanically. The biggest barrier to unguided first use is a structural contradiction introduced by the 2026-07-09 "UX scrub" (commit `5adc7fd`): **all on-screen synthetic-data disclosure was removed and delegated to the presenter's voice, but the artifact's entire reason for existing is to travel without a presenter** (`leavebehind/README.md:7` — "emailed and double-clicked"). An unaccompanied first-time viewer sees "$4.34B" against real FEMA program names and real disaster numbers with nothing on any screen saying the data is synthetic, the 10 PRA questions are placeholders, or that "disbursements" here are calibrated stand-ins for public obligations — three of the package's own non-negotiable guardrails (`11-demo-storyboard.md:5`, `10-risk-assessment-automation.md:154`, ASSUMP-05) that the built UI now violates. The same scrub makes screen 9's transparency claim self-contradicting: the assumptions register silently filters out exactly the assumptions that disclose synthetic provenance. For the analyst persona, the second-order barrier is orientation: the file opens on a results dashboard with no statement of what the tool is, what was done automatically, or that the intended path is a 1→10 walk — and the storyboard's first user action ("upload an extract") has no counterpart or explanation on screen 2, so a first-timer looks for an uploader that doesn't exist. For the exec persona the screens are individually legible, but the AI labels undersell honesty (the on-screen tag says "AI-SUGGESTED" while only the export admits "precomputed for demo"). Almost all of this is recoverable with small, targeted copy and chrome changes; the one structural decision to make is a build-time "presented vs. leave-behind" mode.

## Larger bets

- **B1 — Dual-mode build.** The tension behind UX-01/02/06 is real: the presented demo wants a clean product illusion; the leave-behind must self-disclose. Make disclosure chrome a flag in `build_demo_html.py` (`--presented` strips it; default emits it), producing two artifacts from one template. This preserves the intent of commit `5adc7fd` without shipping an undisclosed-synthetic artifact.
- **B2 — In-file guided tour.** A 10-stop "next →" overlay (one sentence per screen, drawn from the storyboard talking points) would let the exec persona replay the demo alone — the current file has the *content* of `DEMO_SCRIPT.md` but none of its narration. Pure JS/HTML, fits the no-network constraint.
- **B3 — Glossary drawer.** One top-bar "?" opening a short glossary (PRA, PIIA, comprehensive vs preliminary, obligation vs disbursement, DR, exception queue) addresses UX-09 wholesale instead of tooltip-by-tooltip.
- **B4 — Analyst task rail.** For the sign-off job specifically: a small persistent checklist (review answers → answer Q9/Q10 → finalize → export) with live state, bridging screens 7→8→10. This is the "core job with no instructions" feature.
